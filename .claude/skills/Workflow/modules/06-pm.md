# MODULE 06 — PROJECT MANAGER LAYER
# Loaded for: multi-sprint projects

Runs in parallel with the workflow. Does not replace any section.

---

## 1. Stakeholder Communication

At PLAN phase, always identify:
- Decision maker (DM) — who approves this?
- Who is affected by this change?
- Who needs a completion update?

**Rules:**
- Blockers → report immediately, don't wait
- Scope changes → confirm with DM before proceeding. Never silently expand.
- Completion → status format only:

```
STATUS: [DONE / IN PROGRESS / BLOCKED]
DONE: [1-2 lines]
BLOCKER: [none / describe]
NEXT: [1 line]
```

---

## 2. Scope Management

Scope creep = #1 cause of project failure.

When a new request arrives mid-task:
1. STOP — do not add to current work
2. Note in `tasks/backlog.md`
3. Is this in scope? YES (DM confirms) → add to plan, re-verify DoD · NO → park, continue

---

## 3. Prioritization (MoSCoW)
`[M]` Must (blocks delivery) · `[S]` Should (high value) · `[C]` Could (defer) · `[W]` Won't (out of scope)
Apply in `tasks/todo.md`. When competing tasks, always [M] before [S] before [C].

---

## 4. Milestone & Sprint Structure

**Milestone format** (`Docs/ROADMAP.md`):
```
## Milestone N — [Name] — Target: [YYYY-MM-DD]
Goal: [one sentence]
Deliverables: [ ] Item 1  [ ] Item 2
DoD: [how we know it's done]
```

**Sprint rule:** 1 sprint = 1 session.
Start: pick tasks by priority. End: run 04-session §4 (Session End).
Never carry half-done tasks — finish or explicitly park.

---

## 5. Risk Register

For any task > 3 steps, add to plan:
```
RISKS:
- [Risk] → Impact: H/M/L → [Mitigation]
```
Review before EXECUTE. Risk materializes → back to PLAN.

---

## 6. Dependency Tracking

```
Task A → requires Task B (must finish first)
Task C → blocked by external: [describe]
Task D → parallel with Task E (batch in one message)
```
Never start a task with an unresolved hard dependency.

---

## 7. Retrospective (after every milestone)

4 questions → log in `Docs/retro_[YYYY-MM-DD].md`:
1. What went well? → repeat this
2. What went wrong? → fix this
3. What was unclear? → document this
4. What would I do differently? → update skill/workflow

---

## PM Anti-patterns:
- Starting without knowing the DM
- Silently expanding scope
- No backlog — "we'll remember it"
- Skipping risk assessment for "simple" tasks
- Milestone without retro
- Unreported blockers
